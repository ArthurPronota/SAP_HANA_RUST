
__SAP HANA__ — это современная, колоночная база данных, предназначенная для высокопроизводительной обработки транзакций и аналитики в реальном времени. Ее понимание с точки зрения источников данных охватывает четыре ключевых аспекта: уникальный SQL-диалект, богатую систему типов данных, гибкие форматы экспорта и основные задачи администратора.

### 1. SQL-диалект SAP HANA

SQL SAP HANA во многом соответствует стандарту ANSI SQL, но имеет важные особенности, которые необходимо учитывать при работе.

- Колоночный синтаксис: При создании таблиц основным типом является COLUMN TABLE, что оптимизирует хранение и производительность для аналитических запросов.
- Расширения для разработки: Поддерживаются сложные процедурные расширения (SQLScript), позволяющие выполнять сложную бизнес-логику непосредственно на сервере БД.
- Работа с пространственными данными: HANA обладает мощной поддержкой геопространственных данных и функций (ST_Geometry), что нашло отражение в Java-библиотеках для работы с ней.
- Мультимодельность: Позволяет работать с JSON и Graph данными, предоставляя соответствующие операторы для их обработки.

### 2. Типы данных

SAP HANA поддерживает широкий спектр типов данных, которые можно логически разделить на несколько групп. Понимание этих типов критически важно при проектировании схемы данных

|Категория|Типы данных|Описание|
|---------|-----------|--------|
|Символьные|VARCHAR, NVARCHAR, ALPHANUM, SHORTTEXT|Для хранения строк текста. NVARCHAR используется для Unicode.|
|Целочисленные|TINYINT, SMALLINT, INTEGER, BIGINT|Для хранения целых чисел разной размерности.|
|Вещественные|REAL, DOUBLE, FLOAT(p)|Для чисел с плавающей точкой.|
|Десятичные|DECIMAL(p,s), SMALLDECIMAL|Для точных финансовых расчетов с заданной точностью и масштабом.|
|Дата и время|DATE, TIME(p), TIMESTAMP(p), SECONDDATE|Для хранения дат, времени и их комбинаций с точностью до долей секунды.|
|Двоичные|BINARY(n), VARBINARY(n)|Для хранения "сырых" байтовых последовательностей.|
|Большие объекты|BLOB, CLOB, NCLOB, TEXT|Для хранения больших объемов данных, таких как документы или изображения.|
|Логический|BOOLEAN|Для хранения значений TRUE или FALSE.|

### 3. Форматы экспорта данных

SAP HANA предлагает несколько способов экспорта данных, выбор которых зависит от дальнейшего использования данных.

- __CSV__: Это наиболее универсальный и читаемый формат. Идеально подходит для обмена данными с электронными таблицами или другими приложениями, а также для небольших объемов данных.
- __Apache Parquet__: Рекомендуемый формат для больших данных. Обеспечивает высокую степень сжатия и лучшую производительность при аналитике за счет колоночного хранения.
- __Бинарный формат SAP HANA__: Собственный, сжатый формат для создания полных резервных копий схем или целых баз данных.

__Способы экспорта:__
- Wizard в Database Explorer: Предоставляет графический интерфейс для пошагового экспорта данных.
- SQL-оператор EXPORT INTO: Позволяет выполнять экспорт через SQL-скрипты, что удобно для автоматизации. Поддерживает CSV, Parquet и JSON.
- Экспорт каталога: Позволяет выгружать не только данные, но и DDL-скрипты (SQL-код) для воссоздания структуры таблиц, процедур и функций.

### 4. Основы администрации

- Базовая администрация SAP HANA, как правило, выполняется через веб-инструмент SAP HANA Cockpit и инструмент разработчика SAP HANA Database Explorer.
- Управление службами: Администратор может запускать или останавливать как всю базу данных, так и отдельные сервисы (например, indexserver). Операции возможны через Cockpit или SQL-команды ALTER SYSTEM START/STOP DATABASE.
- Управление конфигурацией: Все параметры HANA хранятся в ini-файлах (например, global.ini, indexserver.ini). Их изменение должно производиться только через административные инструменты, после чего часто требуется перезапуск службы.
- Мониторинг: Cockpit предоставляет дашборды для отслеживания ключевых метрик: использование памяти (параметр global_allocation_limit), загрузка CPU, состояние дисков и таблиц.
- Безопасность: Включает в себя управление пользователями и группами, а также назначение ролей и привилегий для контроля доступа к данным.

## Примеры всех sql команд SAP HANA

Команд в SAP HANA сотни, но их можно разделить на 4 ключевые группы. Ниже представлены самые важные примеры для каждой категории, которые чаще всего используются при разработке на Rust или администрировании.

### 1. DDL (Определение данных)

Главное отличие здесь — выбор типа хранилища (COLUMN или ROW).

- Создание таблицы (Column Store — стандарт для HANA):
```sql
CREATE COLUMN TABLE "MY_SCHEMA"."USERS" (
    "ID" BIGINT CS_FIXED PRIMARY KEY,
    "NAME" NVARCHAR(100),
    "JOIN_DATE" DAYDATE DEFAULT CURRENT_DATE
);
```

- Создание индекса:

```sql
CREATE INDEX "IDX_USER_NAME" ON "MY_SCHEMA"."USERS"("NAME");
```

#### 2. DML (Манипуляция данными)

HANA поддерживает стандартный SQL плюс удобные сокращения вроде UPSERT.

- Вставка или обновление (Upsert):

```sql
-- Если ID существует, обновит имя, если нет — вставит строку
UPSERT "MY_SCHEMA"."USERS" ("ID", "NAME") VALUES (1, 'Arthur') WITH PRIMARY KEY;
```

- Чтение с ограничением (HANA-стиль):
```sql
SELECT TOP 10 * FROM "MY_SCHEMA"."USERS" ORDER BY "JOIN_DATE" DESC;
```

- Удаление всех данных (быстрое):
```sql
TRUNCATE TABLE "MY_SCHEMA"."USERS";
```

#### 3. DCL и Административные команды

Эти команды нужны для управления доступом и мониторинга системы.

- Управление правами:
```sql
GRANT SELECT, INSERT ON SCHEMA "MY_SCHEMA" TO "RUST_APP_USER";
```

- Просмотр использования памяти (Админ):

```sql
SELECT * FROM M_SERVICE_MEMORY;
```

- Принудительное создание Savepoint:

```sql
ALTER SYSTEM SAVEPOINT;
```

- Отмена тяжелого соединения:
```sql
ALTER SYSTEM CANCEL SESSION '123456';
```

#### 4. Работа со спецификой HANA (SQLScript)

Это то, что делает HANA мощной — процедуры и табличные переменные.

- Создание процедуры:
```sql
CREATE PROCEDURE "GET_FRIENDS_COUNT" (IN user_id BIGINT, OUT friends_count INT)
LANGUAGE SQLSCRIPT AS
BEGIN
    SELECT COUNT(*) INTO friends_count FROM "FRIENDS" WHERE "USER_ID" = :user_id;
END;
```

- Вызов процедуры из Rust:

```sql
CALL "GET_FRIENDS_COUNT"(1, ?);
```

#### 5. Гео-данные и Текстовый поиск

- Создание нечеткого (Fuzzy) поиска:
```sql
-- Найдет "Arthur", даже если в запросе "Artur"
SELECT * FROM "USERS" WHERE CONTAINS("NAME", 'Artur', FUZZY(0.8));
```

- Работа с координатами:

```sql
SELECT ST_GeomFromText('POINT(55.75 37.61)', 4326).ST_Distance(LOCATION) FROM "STORES";
```
