## Диагностические и трейс-файлы SAP HANA

В терминологии SAP HANA эти понятия часто используют как синонимы, но если разделять их технически, то Трейс-файлы — это составляющая часть Диагностических файлов.

Вот основные отличия:

<a name="q-000001"><h3>1. Трейс-файлы (Trace Files) — «Черный ящик»</h3></a>

Это текстовые файлы (с расширением .trc), которые записывают процесс работы.

- Что это: Потоковая запись событий в реальном времени.
- Назначение: Отладка и поиск причин конкретных ошибок (например, почему не прошел платеж или почему отвалился пользователь).
- Примеры: indexserver.trc, nameserver.trc.
- Как работают: Пишутся постоянно. Уровень детализации можно менять (от Info до Debug).

<a name="q-000002"><h3>2. Диагностические файлы (Diagnostic Files) — «Общий отчет»</h3></a>

Это более широкое понятие, которое включает в себя трейсы, а также снимки состояния системы (Dumps).

- Что это: Набор данных, включающий .trc, .rtedump, .log и конфигурационные файлы.
- Назначение: Полный анализ здоровья системы, особенно после падения (Crash).
- Примеры:
    - Runtime Dumps (.rtedump): Снимок всей памяти и потоков в момент зависания.
    - Configuration files (.ini): Текущие настройки базы.
    - Topology logs: Схемы распределения данных по узлам.
- Как работают: Часто создаются автоматически только в момент критического сбоя или по требованию администратора.

Сравнение для быстрого понимания:

|Характеристика|Трейс-файлы (.trc)|Диагностические дампы (.rtedump)|
|--------------|------------------|--------------------------------|
|тип записи|Непрерывная (как видеорегистратор)|Моментальный снимок (как фото)|
|Когда смотреть|Когда нужно понять хронологию событий.|Когда база уже упала или зависла.|
|Размер|Растут постепенно, самоочищаются.|Могут быть огромными (гигабайты за раз).|
|Читаемость|Легко читаются человеком в текстовом редакторе.|Сложные для чтения (требуют спец. навыков или скриптов).|

### Итог
Если вы ищете, почему в Rust-приложении выскочила ошибка SQL Error, вы открываете Трейс-файл (indexserver.trc). Если же вся база данных перестала отвечать на любые запросы, вы собираете Диагностический пакет (Full System Dump) для отправки в поддержку SAP.
